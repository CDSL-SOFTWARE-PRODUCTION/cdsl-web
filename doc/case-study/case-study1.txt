## 2. Chi Tiết Case Study

### Problem (Vấn đề)
Các phòng khám hiện đại sử dụng nhiều thiết bị chụp ảnh thông thường (máy ảnh kỹ thuật số, smartphone y tế) để ghi lại tình trạng lâm sàng (ví dụ: ảnh da liễu, ảnh nội soi, hoặc văn bản scan).
Tuy nhiên, hệ thống quản lý hình ảnh y tế trung tâm (**PACS**) thường chỉ chấp nhận chuẩn **DICOM**.
*   **Nỗi đau**: Bác sĩ phải mở nhiều phần mềm khác nhau để xem ảnh: một cho X-Quang (DICOM) và một cho ảnh chụp thông thường (JPG/PNG).
*   **Rủi ro**: Dữ liệu hình ảnh rời rạc, khó lưu trữ lâu dài và dễ nhầm lẫn thông tin bệnh nhân do quản lý file thủ công.

### Solution (Giải pháp)
Chúng tôi phát triển **DVT Image-to-DICOM Converter**, một module xử lý trung gian mạnh mẽ:
*   **Input**: Chấp nhận mọi định dạng ảnh phổ biến (JPG, PNG, BMP) từ máy tính hoặc thiết bị di động của bác sĩ.
*   **Processing**: 
    *   Sử dụng Python (`pydicom`, `numpy`) để xử lý ma trận điểm ảnh (pixel data) chính xác.
    *   Tự động map thông tin bệnh nhân (Patient Name, ID) vào metadata của file DICOM (`Tags: 0010,0010`, `0010,0020`).
    *   Hỗ trợ cả ảnh màu (RGB) và ảnh đen trắng (Monochrome) với chuẩn nén tối ưu.
*   **Output**: File chuẩn DICOM (Secondary Capture) tương thích 100% với các hệ thống PACS phổ biến (Orthanc, dcm4chee).

### Result (Kết quả)
Sau 3 tháng triển khai tại phòng khám thí điểm:
*   Đã xử lý thành công hơn **10.000+ hình ảnh** lâm sàng.
*   Loại bỏ hoàn toàn quy trình "in ảnh kẹp hồ sơ" giấy truyền thống.
*   Tốc độ truy xuất hồ sơ bệnh nhân tăng gấp **2 lần** nhờ dữ liệu tập trung.

## 4. Hướng dẫn sử dụng thử (Technical Demo)
*Dành cho team kỹ thuật muốn kiểm tra năng lực xử lý:*
1.  Chạy script: `python3 test.py`
2.  Input: File `test.jpg` (bất kỳ ảnh nào).
3.  Output: File `output.dcm` được sinh ra với đầy đủ Tags (PatientID: 123456, Modality: OT).
4.  Mở `output.dcm` bằng bất kỳ DICOM Viewer nào để kiểm chứng.
