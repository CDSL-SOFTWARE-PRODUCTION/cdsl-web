## 2. Case Study Chi Tiết

### Problem (Vấn đề)
> *Thách thức xử lý Big Data dạng text phi cấu trúc.*
*   **Bối cảnh:** Khách hàng phát triển App học tiếng Trung, cần import bộ từ điển nguồn mở CC-CEDICT (hơn 100MB text, >100.000 mục từ) vào hệ thống Database.
*   **Nỗi đau của khách:** Format file gốc `.txt` rất phức tạp (lẫn lộn Phồn thể, Giản thể, Pinyin trong ngoặc, và định nghĩa).
*   **Hạn chế cũ:** Các tool convert online bị crash do file quá nặng. Team Content thử mở bằng Excel thì bị vỡ font và treo máy. Dự kiến nhập tay hoặc copy-paste sẽ mất cả tháng và dễ sai sót.

### Solution (Giải pháp)
> *Tiếp cận "Divide and Conquer" (Chia để trị) với Python Automation.*
*   **Kiến trúc xử lý dòng (Stream Processing):** Thay vì load cả file 100MB vào RAM, tôi viết script `split_cedict.py` để đọc và xé nhỏ dữ liệu thành các "chunks" (10.000 từ/file), đảm bảo RAM luôn thấp (<50MB).
*   **Regex Engine tùy chỉnh:** Xây dựng biểu thức chính quy (Regex) phức tạp để "bắt" chính xác 4 trường dữ liệu:
    *   `Phồn thể` & `Giản thể` (Unicode characters).
    *   `Pinyin` (nằm trong `[]`).
    *   `Định nghĩa` (phân tách bởi `/`).
*   **Data Cleaning:** Tự động chuẩn hóa văn bản (Clean Text), chuyển đổi ký tự phân cách thành xuống dòng (`\n`) để hiển thị đẹp mắt trong Excel.

### Result (Kết quả)
> *Biến "cục đá" dữ liệu thô thành "viên ngọc" thông tin.*
*   Hệ thống chạy ổn định, output ra 10+ file Excel được định dạng chuẩn chỉ.
*   Dữ liệu sau khi xử lý có thể import trực tiếp vào SQL/CMS của App mà không cần sửa tay.
